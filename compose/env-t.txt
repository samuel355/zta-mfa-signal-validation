1) Spoofing-Heavy (geo/Wi-Fi mismatch)
# Class balance
SIM_ATTACK_ONLY=false
SIM_BENIGN_KEEP=0.30
SIM_MAX_PER_FILE=500

# Spoofing knobs
SIM_INJECT_GPS_MISMATCH=0.60   # ↑ mismatch rate
SIM_GPS_OFFSET_KM=800          # push far enough to pass your 50km check
SIM_USE_GPS_FROM_WIFI=true

# De-emphasize posture
SIM_PATCHED_TRUE_RATE=0.90     # most devices patched -> fewer POSTURE_OUTDATED

# TLS rare bad
SIM_TLS_BAD_RATE=0.05

Expected:
	•	validated_context.cross_checks.gps_wifi_far=true often.
	•	reasons dominated by GPS_MISMATCH, WIFI_MISMATCH.
	•	SIEM stride ≈ Spoofing (most), some Tampering.
	•	Risk around 0.5 on spoofing (→ many step_up), some allow.

	# Class balance

3) DoS/Brute-Force Focus (CICIDS attacks only)
# Only attacks, pick families
SIM_ATTACK_ONLY=true
SIM_ATTACK_WHITELIST=DDOS,PORTSCAN,HEARTBLEED,INFILTRATION,BOT,WEB_ATTACKS
SIM_MAX_PER_FILE=800

# Reduce benign noise
SIM_BENIGN_KEEP=0.00

# Turn off synthetic spoofing
SIM_INJECT_GPS_MISMATCH=0.0
SIM_USE_GPS_FROM_WIFI=false

# Kill posture entirely
SIM_PATCHED_TRUE_RATE=1.0

# TLS anomalies only when label implies (keep small)
SIM_TLS_BAD_RATE=0.05

Expected:
	•	Labels map to reasons:
	•	DDOS/DoS/PortScan → BRUTE_FORCE → STRIDE DoS
	•	Web Attacks/SQLi/XSS → POLICY_ELEVATION → EoP
	•	Bot/Infiltration → DOWNLOAD_EXFIL → InformationDisclosure
	•	Heartbleed → TLS_ANOMALY → Tampering
	•	Risk often 0.5 (single strong reason) → many step_up. With SIEM aggregation you’ll see some deny.
